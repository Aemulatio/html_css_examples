# CSS Container Queries: практическое объяснение

## Зачем нужны
`@media` смотрит на viewport (все окно), а `@container` смотрит на размер конкретного контейнера.
Это ключевая фича для переиспользуемых компонентов: карточек, виджетов, сайдбаров, блоков в CMS.

## Базовый синтаксис
```css
.host {
  container-name: cards;
  container-type: inline-size;
}

@container cards (min-width: 520px) {
  .cards { grid-template-columns: 1fr 1fr; }
}
```

## Важные правила
1. Без `container-type` контейнерные запросы не сработают.
2. `inline-size` почти всегда лучший старт: реагирует на ширину.
3. Лучше задавать `container-name`, если в дереве много контейнеров.
4. Пиши брейкпоинты от компонента, а не от экрана.

## Когда использовать
- Один и тот же компонент должен работать в основной колонке и в узком сайдбаре.
- Компонент в модалке имеет другой размер, чем на странице.
- Нужна библиотека UI-компонентов без привязки к layout проекта.

## Антипаттерны
- Использовать `@container` там, где достаточно `@media` для всей страницы.
- Слишком много мелких брейкпоинтов (сложно сопровождать).
- Забывать fallback для старых браузеров.

## Fallback-стратегия
- Сначала делай простой одно-колоночный базовый layout.
- Улучшай `@container`-правилами как progressive enhancement.
- Проверяй поведение в узком/среднем/широком контейнере отдельно от viewport.

## Что смотреть в демо
Файл: `pages/container-queries/index.html`
- Контейнер `resize: horizontal` позволяет руками менять размер.
- На `520px` карточки переходят в 2 колонки.
- На `760px` карточки переходят в 3 колонки.
